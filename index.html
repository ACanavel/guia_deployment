Guía de Deployment en Netlify

Inicio .gitignore Conclusión

¿Qué es Netlify y por qué usarlo?

Netlify es una plataforma que permite alojar y desplegar fácilmente aplicaciones web estáticas o con algunas funcionalidades de backend (serverless). Ofrece integración con repositorios (GitHub, GitLab, Bitbucket), despliegues automáticos, manejo sencillo de dominios y certificados SSL, y más.

¿Qué es filess.io?

filess.io es un servicio gratuito para subir y compartir archivos de manera sencilla, generando un enlace de descarga que puedes usar en tu aplicación. Útil para archivos grandes o recursos que no desees alojar directamente en tu repositorio de GitHub, en este caso la base de datos.

Flujo general de la guía

  1. Subir tu proyecto a GitHub.
  2. Subir archivos grandes a filess.io.(Aca subiras tu base de datos, te van a generar variables de entorno que tendras que sustituir en tu proyecto)
  3. Conectar GitHub con Netlify.
  4. Configurar el despliegue en Netlify.
  5. Probar la aplicación.
  6. (Opcional) Añadir dominio personalizado.

1. Subir tu proyecto a GitHub

Crea un repositorio en GitHub y luego, en tu carpeta local, ejecuta: 

git init
git add .
git commit -m "Primer commit"
git remote add origin https://github.com/<TU_USUARIO>/<TU_REPO>.git
git push -u origin main

Esto conectará tu proyecto local con GitHub. Ajusta la rama principal (main o master) según corresponda.

2. Subir archivos a filess.io

  1. Visita filess.io y crea o inicia sesión en tu cuenta.
  2. Sube los archivos que desees compartir (imágenes, videos, etc.).
  3. filess.io generará un enlace que podrás usar en tu código.

3. Conectar GitHub con Netlify

  1. Crea una cuenta en Netlify.
  2. En tu panel, selecciona "New site from Git".
  3. Autoriza a Netlify para acceder a tu cuenta de GitHub.
  4. Elige tu repositorio donde está el proyecto.

4. Configurar despliegue en Netlify

  • Selecciona la rama: main o master.
  • Comando de build: (Por ejemplo, npm run build para React).
  • Carpeta de publicación: build, dist, etc.
  • (Opcional) Variables de entorno: Configúralas en Site Settings > Build & Deploy > Environment.
  • Haz clic en Deploy site.

5. Verificar tu sitio

Netlify te dará una URL (por ejemplo https://mi-sitio.netlify.app). Visítala y comprueba que tu proyecto funcione y que los enlaces a filess.io estén correctos.

.gitignore y variables de entorno

Crea el archivo .gitignore en la raíz de tu proyecto para excluir archivos o carpetas que no quieras subir. Por ejemplo: 

node_modules/
.env

Para manejar variables de entorno en Netlify, ve a Site Settings > Build & Deploy > Environment y defínelas allí.

6. (Opcional) Dominio personalizado

Si tienes un dominio propio, configúralo en la sección Domain Settings de Netlify y apunta los registros DNS. Netlify gestionará el certificado SSL.

Conclusión

¡Listo! Con esta guía podrás desplegar tu proyecto en Netlify usando GitHub y filess.io. Revisa los logs de Netlify si hay errores y asegúrate de que tus rutas a filess.io estén bien configuradas. ¡Éxitos!
